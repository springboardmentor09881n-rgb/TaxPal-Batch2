# 🔀 Git Workflow

1. Every team member works on their own Git branch.
2. Changes are pushed only to individual branches.
3. Pull Requests are reviewed before merging into `main`.
4. The `main` branch always contains the latest stable implementation.

---

# 📝 Notes

- The project follows a modular architecture with separate frontend, backend, and database components.
- Parallel development is being followed to improve development efficiency.
- MongoDB Atlas is used as the cloud database.
- Angular is used for frontend development.
- Node.js and Express.js are used for backend API development.
- Regular code reviews and documentation updates are maintained throughout the project.
